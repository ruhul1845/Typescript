# why unknown is safe than any 

 `any` and `unknown` are commonly used  Although they may look similar, they behave very differently.

The `any` type removes TypeScript’s type checking completely, while `unknown` keeps type safety and push  developers to verify data before using it.

---

## Why `any` is Called a Type Safety Hole

The `any` type disables all type checking. TypeScript allows any operation on a variable with the `any` type, 
even it is invalid thats why any is called safety hole
